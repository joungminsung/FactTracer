# FactTracer Persona Web QA Report

검증일: 2026-06-10  
주요 대상: 현재 개발 서버 `http://localhost:3002`  
보조 확인: Docker 통합 서버 `http://localhost:3000`, API `http://localhost:8000`  
기준 페르소나: 일반 뉴스 사용자, 정치/사회 이슈 관심 사용자, 학생/토론/발표 사용자, 기자/리서처, 관리자/검토자

## 제품 방향 전제

현재 FactTracer의 우선순위는 단일 주장 팩트체크보다 **정확한 보도와 사건 전말 정리**다. 사용자는 사건 흐름, 핵심 사실, 이해관계자별 주장, 의견 충돌, 근거와 보도 차이를 한 번에 파악해야 한다. 팩트체크는 독립 목적이라기보다 정확한 보도를 가능하게 하는 신뢰 장치로 본다.

## 결론

현재 개발 서버의 읽기 화면은 전반적으로 접근 가능하다. 홈, 이슈 상세, 리포트, 결측 상태, 관리자 기본 경로는 사용자 언어로 렌더링되고, `npm run acceptance:audit`도 `localhost:3002` 기준 통과했다.

다만 사용자 행동이 필요한 저장, 검증 요청, 리포트 생성, 인증류 플로우는 정상 QA 완료로 볼 수 없다. `localhost:3002`는 API CORS 허용 origin에 포함되지 않아 브라우저 클라이언트 요청이 `Failed to fetch`로 실패한다. Docker 통합 환경에서는 `localhost:3000` origin은 허용되지만 SQLite 쓰기 잠금으로 인증/리포트 생성에서 `database is locked`가 발생했다.

UI/UX, 사용자 흐름, 디자인, 화면 레이아웃 감사는 별도 문서 `qa/web-qa-2026-06-10/uiux-design-flow-audit.md`에 정리했다.

## 자동 검증

- `FACTTRACER_AUDIT_BASE_URL=http://localhost:3002 npm run acceptance:audit`: 통과, 15개 라우트.
- `FACTTRACER_AUDIT_BASE_URL=http://localhost:3000 npm run acceptance:audit`: 실패. Docker 웹 이미지가 현재 코드보다 오래되어 홈 기대 문구와 관리자 상세 기대 문구가 불일치.
- `GET http://localhost:8000/health`: 정상.
- `OPTIONS Origin: http://localhost:3002`: `400 Disallowed CORS origin`.
- `OPTIONS Origin: http://localhost:3000`: `200 OK`.

## 주요 발견

### P0. 현재 dev 서버의 클라이언트 액션 실패

영향: 저장, 자료 제출/검증 요청, 리포트 생성, 회원가입/로그인 등 브라우저에서 API로 쓰는 액션.

증거:
- `localhost:3002` 이슈 상세에서 `저장` 클릭 시 `Failed to fetch` 노출.
- `localhost:3002/verify`에서 링크 검증 요청 제출 시 `Failed to fetch` 노출.
- API preflight 확인 결과 `http://localhost:3002` origin이 CORS에서 거부됨.

사용자 영향:
- 일반 사용자는 저장/알림 연결을 완료하지 못한다.
- 학생/리서처는 리포트 생성을 완료하지 못한다.
- 모든 페르소나가 오류 원인을 이해할 수 없다.

### P0. Docker 통합 환경 DB 쓰기 잠금

영향: 인증, 리포트 생성, 일부 쓰기 API.

증거:
- API 로그에서 `sqlite3.OperationalError: database is locked`.
- `/v1/auth/login`, `/v1/auth/signup`, `/v1/issues/{issueId}/report` 쓰기 중 발생.
- 브라우저에서는 `처리 시간이 길어지고 있습니다. 잠시 후 다시 시도해 주세요.` 또는 내부 오류로 보임.

### P1. 헤더 검색이 동작하지 않음

`localhost:3002` 헤더 검색 입력창에 `부동산`을 입력하고 Enter를 눌러도 URL이 `/?q=...`로 바뀌지 않고 결과도 갱신되지 않았다. 입력값은 유지되므로 폼 제출 이벤트가 사용자 기대대로 작동하지 않는 상태다.

### P1. 분야 필터 상태에서 카테고리 카드 제목과 내용 불일치

`/?topic=경제`로 이동하면 주요 사건은 경제 이슈 중심으로 보이지만, `분야별 사건` 영역에서 `정치`, `사회` 같은 고정 제목 아래 경제 이슈가 표시된다. 필터링된 배열을 고정 카테고리 슬롯에 매핑하면서 레이블이 어긋난 것으로 보인다.

### P1. 사용자 오류 메시지가 기술적임

현재 dev 서버에서는 저장/검증 요청 실패가 `Failed to fetch`로 그대로 노출된다. 이는 CORS 문제의 결과지만 사용자는 로그인 필요, 연결 실패, 서버 지연 중 무엇인지 알 수 없다.

### P1. 리포트/타임라인에 내부 이벤트명 노출

직접 리포트 URL 접근은 가능하지만 본문에 `new_article` 내부 이벤트명이 노출된다. 정확한 보도 리포트 관점에서는 `새 기사 반영`, `새 국면 반영`, `판정 변경` 같은 편집 언어가 적합하다.

### P2. 홈 주요 사건 카드에 빈 링크가 중복 노출

주요 사건 카드에서 같은 이슈 링크가 이미지/썸네일 영역과 텍스트 영역으로 나뉘어 잡히며, 빈 accessible name 링크가 존재한다. 키보드/스크린리더 사용자는 목적을 알 수 없는 링크를 만날 수 있다.

### P2. 리포트 복사 액션은 성공 피드백이 없음

인용 복사는 클립보드에 실제 텍스트를 넣는 것을 확인했다. 다만 화면에 “복사됨” 피드백이 없어 사용자가 성공 여부를 확인하기 어렵다.

## 페르소나별 결과

### 일반 뉴스 사용자

가능:
- 홈에서 주요 사건과 최신 정리 흐름을 빠르게 훑을 수 있다.
- 분야 링크로 관심 주제 진입이 가능하다.
- 이슈 상세에서 핵심 요약, 사건 흐름, 쟁점, 관련 출처를 볼 수 있다.

문제:
- 저장이 실패하고 `Failed to fetch`만 보인다.
- 헤더 검색이 동작하지 않는다.
- 분야 필터 후 카테고리 레이블이 틀린 정보를 줄 수 있다.

### 정치/사회 이슈 관심 사용자

가능:
- 상세에서 쟁점 지도, 관점별 주장, 주장 검증 섹션을 탐색할 수 있다.
- 낙인 표현보다 사건 전말, 주장, 근거 중심 구조가 전면에 있다.

문제:
- 제보/정정 등 쓰기 액션은 CORS/DB 문제에 막힐 가능성이 높다.
- “공식 근거 확인 전”, “초기 기준”이 반복되어 보도 완성도와 근거 연결도가 낮아 보일 수 있다.

### 학생/토론/발표 사용자

가능:
- `/reports/{issueId}` 직접 접근 시 사건 전말과 근거를 재사용할 수 있는 발표/연구용 리포트 구조가 제공된다.
- PDF 저장, 인용 복사, 비교표 복사, 출처 목록 복사 버튼이 보인다.

문제:
- 상세의 `리포트 생성` 액션은 현재 실패한다.
- 리포트 본문에 `new_article` 같은 내부 이벤트명이 남아 있다.
- 복사 성공 피드백이 없다.

### 기자/리서처/콘텐츠 제작자

가능:
- 기사 비교표, 원문 자료, 출처 목록, 판정 변화와 사건 업데이트 흐름을 확인할 수 있다.
- 표는 모바일/데스크톱 모두 페이지 전체 폭을 밀지 않고 내부 스크롤로 처리된다.

문제:
- 공식 근거 연결이 `공식 근거 확인 전`으로 반복되어 인용 가능한 근거 선별이 어렵다.
- 원문 링크가 여러 섹션에 반복되어 리포트가 길어지고 중복 출처 정리가 필요하다.

### 관리자/검토자

가능:
- `/admin`, `/admin/reports`, `/admin/sources`, `/admin/agents`, `/admin/settings`, `/admin/issues/ISS-260608-001`가 사용자 언어로 렌더링된다.
- 자동 감사 기준 문구도 현재 dev 서버에서 통과한다.

문제:
- 비로그인 상태에서도 일부 관리자 하위 페이지의 입력 폼이 보인다.
- 운영 액션을 실제 제출할 경우 CORS/DB 쓰기 문제 영향을 받을 가능성이 높다.

## 반응형 확인

- `390x844` 모바일 기준 홈과 상세 모두 페이지 전체 가로 스크롤은 없었다.
- 하단 모바일 탭이 노출된다.
- 상세 표는 내부 가로 스크롤 컨테이너로 처리된다.
- 모바일 헤더에는 검색/인증 텍스트가 축약되어 주요 탐색은 분야 nav와 하단 탭에 의존한다.

## 권장 수정 순서

1. `FACTTRACER_CORS_ORIGINS`에 현재 dev origin인 `http://localhost:3002`와 `http://127.0.0.1:3002`를 추가한다.
2. Docker SQLite 쓰기 잠금 원인을 분리한다. 내장 스케줄러/워커와 사용자 요청이 같은 SQLite 파일을 쓰는 경합이 핵심으로 보인다.
3. 클라이언트 API 오류를 `Failed to fetch` 대신 사용자 메시지로 매핑한다.
4. 헤더 검색 제출 동작을 고친다.
5. `?topic=...` 상태에서 분야별 사건 레이블/데이터 매핑을 고친다.
6. 리포트/타임라인의 내부 이벤트명을 보도 편집 언어로 변환한다.
7. 주요 사건 카드의 빈 링크 접근성을 개선한다.
8. 복사/저장/자료 제출/검증 요청 성공 상태를 화면에 남긴다.
